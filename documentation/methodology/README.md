# QUAD Methodology Documentation

## Document Structure

This folder contains the complete QUAD (Quick Unified Agentic Development) methodology documentation.

### Primary Document

| Document | Description |
|----------|-------------|
| **[QUAD.md](QUAD.md)** | **MAIN DOCUMENT** - The complete, authoritative QUAD methodology. Contains all sections, diagrams, and detailed specifications. This is the source of truth. |

### Supplementary Documents

These documents are derived from QUAD.md for specific purposes:

| Document | Purpose | Audience |
|----------|---------|----------|
| [QUAD_SUMMARY.md](QUAD_SUMMARY.md) | High-level overview (2-3 page read) | Executives, new team members, quick reference |
| [QUAD_JARGONS.md](QUAD_JARGONS.md) | Terminology glossary and quick reference cards | Anyone needing term definitions |
| [QUAD_DETAILS.md](QUAD_DETAILS.md) | Technical deep-dive on specific topics | Architects, Tech Leads implementing QUAD |
| [QUAD_CASE_STUDY.md](QUAD_CASE_STUDY.md) | Calculator App: Agile vs QUAD comparison | Teams evaluating QUAD adoption |

### Reading Order

**For new readers:**
1. Start with **QUAD_SUMMARY.md** (10 min read)
2. Read **QUAD.md** for full understanding (30-45 min read)
3. Reference **QUAD_JARGONS.md** for terminology

**For implementers:**
1. Read **QUAD.md** completely
2. Use **QUAD_DETAILS.md** for technical specifications
3. Review **QUAD_CASE_STUDY.md** to understand practical application

**For quick reference:**
- **QUAD_JARGONS.md** - Term definitions
- **QUAD_SUMMARY.md** - Core concepts reminder
